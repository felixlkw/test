# SafeMate 중공업 TBM 수동 테스트 시나리오

**대상**: 비개발자(현장 안전관리자, PM, 데모 진행자)
**소요 시간**: 약 22~28분 (사진 분석·PDF 다운로드 포함)
**준비물**: 노트북 또는 스마트폰 1대, 마이크, 카메라, 인터넷 연결, 백엔드 가동 중

> **테스트 콘셉트** — 가상의 조선소 야드에서 오전 **밀폐공간 작업(블록 내부 도장 점검)** TBM을 진행한다. 작업자 3명, 블록 탱크 내부 진입 예정, **밀폐공간(confined_space=true)** 조건에 더해 공기 중 잔류 유기용제 가능성. 인접 야드에서 지난주 산소 결핍 의식 상실 사례가 있었다는 설정. 이 시나리오로 앱의 모든 주요 기능(작업 선택 → 밀폐공간 conditional → 음성 대화 → 사진 분석 → 정리·서명 → PDF 발행 → 보관함 확인)을 끝까지 사용한다.

> **중공업이 다른 도메인과 차별되는 포인트**
> 1. **밀폐공간·SCBA 흐름** — 산소 19.5~23.5%, LEL 10% 미만, 황화수소 10ppm 미만 등 **수치 기반 검증**이 baseline에 명시.
> 2. **양중작업 풍속 임계** — 옥외 야드 특성상 풍속 15 m/s 이상이면 양중 중단(LIFT-W01).
> 3. **화기작업 잔불 점검 1시간** — 야드 강풍에 의한 비산 위험 때문에 건설(30분)보다 길게 잡혀 있음.
> 4. **2인 1조·감시인** — 단독 작업 금지가 명문화되어 음성 대화에서 자연스럽게 묻고 답하기.

## 0. 시작 전 점검 (1분)

1. **백엔드 가동 확인** — `http://localhost:8000/api/health` → `ok`.
2. **앱 열기** — `http://localhost:5173` → SafeMate 헤더.
3. **권한 미리 허용** — 마이크/카메라 둘 다 **허용**.

## 2. 준비 화면 — 작업 선택 + AI 위험 추천 (5분)

### 2-1. 작업 유형 선택

**할 일**
- 3개 카드(양중작업 / 화기작업 / 밀폐공간 작업) 중 **"밀폐공간 작업"** 을 탭.

**통과 기준**
- [ ] 카드 테두리 주황색 강조.
- [ ] 위험 추천 1~3초 안에 채워짐.

### 2-2. 현장 컨텍스트 입력

**할 일** — 6개 필드 입력:

| 필드 | 입력값 |
|---|---|
| 작업자 수 | **3** |
| 교대 | **주간** |
| 풍속 (m/s) | (비워둠 — 옥내 블록 내부) |
| 신규 자재 | **유기용제 도료** |
| 특이사항 | **블록 내부 도장 점검, 인접 야드에서 산소 결핍 사고 1주 전 발생** |
| 과거 사고 키워드 | **산소 결핍, 의식 상실, H2S** |

**통과 기준**
- [ ] AI 컨텍스트 폼 **활성화**(중공업 default ON).
- [ ] 모든 필드 정상 입력.

### 2-3. AI 위험 추천 결과 + "다시 받기"

**할 일** — **🔄 다시 받기** 1회.

**통과 기준 — 필수 점검(자물쇠 🔒)**
- [ ] 5개 baseline 항목 자동 펼침 (CS-01~05).
- [ ] 예시:
  - "밀폐공간 작업허가서 발급 및 위험요인(질식/화재/유해가스) 명기"
  - "산소(19.5~23.5%)·가연성가스(LEL 10% 미만)·황화수소(10ppm 미만) 측정"
  - "송풍기 또는 강제환기 가동 및 작업 중 지속 환기"
  - "감시인 배치 및 통신수단(무전기) 확보"
  - "긴급구조용 SCBA(공기호흡기)·구명로프 비치"
- [ ] 각 항목에 산업안전보건기준규칙 §619 / §621 / §624 등 법규 인용 + 출처 칩.

**통과 기준 — 컨텍스트 반영**
- [ ] 특이사항(블록 내부 도장)·과거 사고(H2S, 산소 결핍)가 추천 질문이나 조건부 항목에 반영된다.
- [ ] 가능하면 LLM이 동적으로 "유기용제 도료 — 휘발성 농도 측정 추가" 같은 조건부를 생성한다.

### 2-4. 추천 질문 확인

**통과 기준**
- [ ] 4~6개 추천 질문 칩.
- [ ] 예: "오늘 4-가스 검지기 충전 상태 확인하셨습니까?", "감시인 무전 채널 통화 시험 마쳤나요?", "구조자 SCBA 사전 훈련 받으셨습니까?"

### 2-5. "TBM 시작" 진입

**할 일** — **"TBM 시작"** 탭.

**통과 기준**
- [ ] `/tbm/.../run` 진입.
- [ ] 자동 음성 세션 연결.

## 3. 음성 대화로 TBM 진행 (10~12분)

### 3-1. 첫 응답 + 사전 정보

**할 일** — 마이크 ON 후 발언:
> "오늘 블록 도장 점검 작업입니다. 작업자 3명, 블록 내부 진입 예정이고, 유기용제 도료 사용해요. 인접 야드에서 1주 전 산소 결핍으로 의식 잃은 분이 계셨거든요."

**통과 기준**
- [ ] AI가 **밀폐공간 + 유기용제 + 과거 H2S 사고**를 인지하고 자연스럽게 첫 발화에 반영.
- [ ] AI가 4-가스 검지기, SCBA, 감시인 배치를 우선적으로 거론.

### 3-2. 체크리스트 단계 진입 — 수치 기반 검증

**할 일** — 5개 baseline 항목 음성 응답:
- "작업허가서 발급했고, 위험요인 3종 다 명기했습니다." → ✅ CS-01
- "4-가스 검지기로 측정했어요. 산소 20.8%, LEL 0%, H2S 1ppm, CO 0ppm입니다." → ✅ CS-02
- "송풍기 가동 중이고, 작업 종료까지 환기 유지할 예정이에요." → ✅ CS-03
- "감시인 1명 입구 상주, 무전 5분 간격 호출 시험 통과했습니다." → ✅ CS-04
- "SCBA 1조 + 구명로프 입구에 비치돼 있고, 구조자 훈련 끝났습니다." → ✅ CS-05

**통과 기준**
- [ ] 게이지 0/5 → 5/5.
- [ ] AI가 산소 농도 20.8%를 듣고 "기준(19.5~23.5%) 안쪽이라 안전합니다" 같이 **수치 검증**을 한다.
- [ ] 단계 전환 시점에 누락 위험 점검(예: "유기용제 휘발 농도는 별도 측정하셨나요?")이 1회 등장하고 거절 시 양보.

### 3-3. 사진 분석으로 SCBA·검지기 점검

**할 일** — 카메라 → 셔터 → SCBA·검지기·환기설비 사진 캡처(없으면 대체 사진).

**통과 기준**
- [ ] AI가 사진 내 **호흡장비·검지기·환기관** 같은 요소를 검출.
- [ ] bbox + 신뢰도 카드 inline 노출.
- [ ] 신뢰도 0.7 이상 자동 보강.
- [ ] **AI가 음성으로도 vision 결과를 1~2문장 설명**(사전정보·점검 단계 한정). auto-boost 발동 시 "체크리스트에 자동 추가했습니다" 자연 언급.

### 3-4. 양중작업 모드로 잠깐 전환 (옵션 — 풍속 conditional 검증)

**할 일** — 추가 발언:
> "오후에 블록을 야드 밖으로 양중할 예정인데, 풍속 16 m/s 예보면 어떻게 해야 하나요?"

**통과 기준**
- [ ] AI가 **"풍속 15 m/s 이상이면 양중 중단"** (LIFT-W01)을 명확히 안내.
- [ ] 스톰핀 체결, 출입금지 구역 설정 같은 항목을 거론.
- [ ] 채팅 로그에 안전 경고 시스템 메시지가 1회 push되거나 일반 응답으로 처리.

### 3-5. 정리본 미리보기

**할 일** — 우측 상단 **"정리본 NN%"** 탭.

**통과 기준**
- [ ] **위험요인** 에 baseline 5건 + 사진 자동 추가.
- [ ] **시나리오** 에 "허가서 위험요인 누락…", "산소 결핍 미감지 진입 → 질식 의식 상실" 등.
- [ ] **대응** 에 "위험요인 3종 별도 라인 명기", "휴대용 4-가스 검지기로 진입 직전 + 1시간 간격 측정" 등.
- [ ] **보호구** 에 "휴대용 4-가스 검지기", "SCBA(공기호흡기)", "방수 무전기" 등.
- [ ] **변화** 항목에 "신규 자재: 유기용제 도료 / 과거 사고 키워드: 산소 결핍, 의식 상실, H2S".

### 3-6. (선택) 안전 경고 토스트

**할 일**:
> "감시인이 잠깐 화장실 가는데, 그 사이 작업해도 될까요?"

**통과 기준**
- [ ] **"⚠️ 잠깐만요!"** 토스트 등장.
- [ ] AI가 "감시인 부재 시 즉시 작업 중단, 감시인 복귀까지 대기" 명확히 안내.
- [ ] 채팅 로그 영구 보존.

## 10. 자주 묻는 의문 (테스트 진행자용)

**Q. 산소 농도를 19.0%라고 답하면 어떻게 되나요?**
→ AI가 "기준 19.5% 미만이라 즉시 작업 중단·환기 후 재측정" 같은 안전 경고를 띄워야 정상. 만약 그냥 ✅로 넘어가면 prompt 회귀 가능성 — 보고.

**Q. SCBA 사진을 찍었는데 안 잡혀요.**
→ "특이 위험 미식별" 카드 또는 신뢰도 0.7 미만 항목은 자동 보강 안 함. 정리본에서 수동 추가.

**Q. 양중작업과 밀폐공간을 한 세션에서 다루려면?**
→ 한 세션에서 작업유형은 1개만 선택. 양중작업 본격 검증은 새 TBM(중공업 → 양중작업)으로 진입. 이번 시나리오에선 음성 잠깐 묻기만.

**Q. PDF 한글이 깨져요.**
→ 2026-05-07 다단 fallback 적용 후 정상 동작이 표준. 브라우저에서 깨지면 **DevTools(F12) → Console** 의 `[pdfGenerate]` 로그로 단계별 진단(fetch / embed / magic / 자동 폴백). 자세한 분기 해석은 `manual_test_scenario_manufacturing.md` §10 FAQ 참조.

**Q. 화기작업까지 같이 하려면?**
→ 새 TBM(중공업 → 화기작업)으로 진입. 야드 잔불 1시간 점검·LEL 측정·송기마스크가 baseline에 등장.
