# SafeMate 제조업 TBM 수동 테스트 시나리오

**대상**: 비개발자(현장 안전관리자, PM, 데모 진행자)
**소요 시간**: 약 20~25분 (사진 분석·PDF 다운로드 포함)
**준비물**: 노트북 또는 스마트폰 1대, 마이크, 카메라, 인터넷 연결, 백엔드 가동 중

> **테스트 콘셉트** — 가상의 제조 공장에서 오늘 아침 **프레스 작업** TBM을 진행한다. 작업자 4명이 새로 도입된 알루미늄 자재를 처리할 예정이고, 지난주 인근 라인에서 협착 사고가 있었다는 설정. 이 한 가지 시나리오로 앱의 모든 주요 기능(작업 선택 → AI 위험 추천 → 음성 대화 → 사진 분석 → 정리·서명 → PDF 발행 → 보관함 확인)을 끝까지 사용한다.

---

## 0. 시작 전 점검 (1분)

1. **백엔드 가동 확인**
   - 별도 터미널에서 `cd backend && uvicorn src.main:app --reload` 실행되어 있어야 함.
   - 브라우저 주소창 `http://localhost:8000/api/health` → `{"status":"ok",...}` 응답 보이면 통과.
2. **앱 열기**
   - 새 터미널에서 `cd frontend && npm run dev` → `http://localhost:5173` 접속.
   - 화면 상단에 PwC 로고 + 검정 글씨 **SafeMate** 가 보이면 OK.
3. **권한 미리 허용**
   - 브라우저 주소창 자물쇠 아이콘 → 마이크 **허용**, 카메라 **허용**.
   - 처음 보는 사용자는 "녹음 안 되는데요?" 라며 당황하기 쉬워서 미리 풀어두는 편이 매끄럽다.

## 1. 홈 화면에서 새 TBM 시작 (1분)

**할 일**
- 메인 화면 가운데 **"새 TBM 시작"** 큰 주황색 버튼 클릭.
- 아래에서 위로 올라오는 "산업 도메인 선택" 시트에서 **"제조"** 카드 탭.

**보이는 것 / 통과 기준**
- [ ] 도메인 선택 시트에 4개 카드(제조 / 건설 / 중공업 / 반도체)가 2×2 격자로 보인다.
- [ ] "제조" 탭 후 곧바로 **"준비"** 라는 제목의 새 화면(`/tbm/.../prepare`)으로 이동한다.
- [ ] 이전에 진행 중이던 세션이 있으면 홈 화면 상단에 주황색 **"진행 중"** 카드가 떠 있을 수 있음 — 무시하고 새 TBM을 시작한다.

> **팁** — 만약 앞서 설정에서 "기본 도메인"을 제조로 지정해 뒀다면, 도메인 시트 없이 곧장 준비 화면으로 점프한다. 이번 테스트에서는 시트 노출을 보기 위해 기본 도메인은 **빈 값** 상태로 둔다(설정 → 운영에서 확인 가능).

## 2. 준비 화면 — 작업 선택 + AI 위험 추천 (4분)

준비 화면은 위에서부터 ① 도메인 표시 → ② 작업 선택 → ③ 현장 컨텍스트 → ④ 위험 추천 → ⑤ 추천 질문 → ⑥ "TBM 시작" 버튼 순서로 흐른다.

### 2-1. 작업 유형 선택

**할 일**
- 화면 가운데 **"작업 선택"** 섹션에서 카드 3개(조립라인 작업 / 지게차 운반작업 / 프레스 작업) 중 **"프레스 작업"** 을 탭한다.

**통과 기준**
- [ ] 선택한 카드 테두리가 주황색으로 강조된다.
- [ ] 선택 즉시 화면 하단으로 자동 스크롤되며 **"위험 추천"** 섹션에 로딩 메시지("위험 추천 불러오는 중…")가 잠깐 떴다 사라진다.
- [ ] 1~3초 안에 추천이 채워진다(GPT-4o 응답 시간).

### 2-2. 현장 컨텍스트 입력

**할 일**
- 작업 선택 바로 아래 펼쳐진 **"현장 상황을 입력하면 AI가 더 정확한 위험을 제안합니다"** 폼에 다음 6개 입력:
  | 필드 | 입력값 |
  |---|---|
  | 작업자 수 | **4** |
  | 교대 | **주간** |
  | 풍속 (m/s) | (비워둠 — 옥내라 무관) |
  | 신규 자재 | **알루미늄 5052 판재** |
  | 특이사항 | **금형 교체 1회 예정** |
  | 과거 사고 키워드 | **협착, 면장갑 말림** (콤마 또는 엔터로 구분) |

**통과 기준**
- [ ] 모든 입력이 잘 받아진다(반도체 도메인이 아니므로 disabled 안 됨).
- [ ] **입력해도 AI 추천이 즉시 다시 호출되지는 않는다** — 키 입력마다 호출되면 5회/60초 제한에 걸리기 때문에 일부러 그렇게 만들었다. 컨텍스트를 반영하려면 다음 단계의 "다시 받기"를 사용한다.

### 2-3. AI 위험 추천 결과 확인 + "다시 받기"

**할 일**
- "위험 추천" 섹션 우측 상단의 **🔄 다시 받기** 버튼을 1회 클릭한다 → 컨텍스트가 반영된 새 추천이 도착한다.

**통과 기준 — 필수 점검(자물쇠 아이콘 🔒)**
- [ ] 5개 항목이 자동으로 펼쳐져 있다 (PR-01 ~ PR-05). 자물쇠 아이콘 = 사용자가 끌 수 없는 필수 항목.
- [ ] 각 항목 하단에 작은 회색 글씨로 **출처 칩** ("catalog" 또는 "llm")과 산업안전보건기준규칙 §103 같은 법규 인용이 보인다.
- [ ] 예시 항목: "프레스 양수기동장치(투핸드) 정상작동 점검", "광전자식 안전장치(라이트커튼) 작동거리·차단 시간 확인", "금형 교체 시 LOTO 시행" 등.

**통과 기준 — 조건부 점검**
- [ ] 컨텍스트(특이사항: 금형 교체)가 반영된 1~2개의 추가 항목이 회색 톤으로 보인다(GPT-4o가 동적으로 생성).
- [ ] 각 조건부 항목 위에 **"if: ..."** 한국어 조건문이 보인다 (예: "if 금형 교체 시" → "교체 직후 시운전은 빈 가동 1회 후 정상 동작 확인").

### 2-4. 추천 질문 확인

**통과 기준**
- [ ] 화면 더 아래에 **"추천 질문"** 4~6개가 칩 형태로 나열된다.
- [ ] 예: "라이트커튼 차단 거리 산정 방식 알고 있습니까?", "오늘 LOTO 락 개인별로 가지고 오셨나요?" 같은 현장 묻기 좋은 질문.
- [ ] 이번 테스트에서는 클릭하지 않는다(다음 화면에서 음성으로 자연스럽게 다룸).

### 2-5. "TBM 시작" 진입

**할 일**
- 화면 맨 아래 주황색 **"TBM 시작"** 버튼 탭.

**통과 기준**
- [ ] 화면이 음성 대화 화면(`/tbm/.../run`)으로 전환된다.
- [ ] 자동으로 AI 음성 세션이 연결된다(상단 상태 점이 회색 → 노란색 "연결 중" → 파란색 "대기").

## 3. 음성 대화로 TBM 진행 (8~10분)

### 3-1. 첫 임프레션 점검

**보이는 것 (위에서부터)**
1. 상단 바: 모드(TBM)/언어(한국어) 칩, 우측에 **"정리본 N%"** 버튼과 **"종료"** 버튼.
2. **단계 스트립** (StagesStrip): 사전 정보 → 점검 → 대응 → 정리 — 현재 단계가 굵은 주황색.
3. **체크리스트 진행 게이지** (예: 0/5).
4. **AI 안내문** ("안녕하세요. 오늘 프레스 작업 TBM을 진행하겠습니다…").
5. **마이크 켜기** 안내 배너 (🎙️ "음성으로 대화하려면 아래 마이크 버튼을 눌러주세요").
6. **InputDock** (하단): 좌측 마이크 버튼 / 가운데 텍스트 입력 / 우측 카메라 버튼.

### 3-2. 마이크 켜고 첫 응답

**할 일**
- 좌측 하단 마이크 버튼 클릭 → 주황색으로 켜짐. 안내 배너는 자동으로 사라진다.
- 다음 문장을 천천히 말한다:
  > "오늘 프레스 작업, 작업자 4명이고요. 신규 자재로 알루미늄 5052 판재 들어옵니다. 금형 교체도 한 번 있어요."

**통과 기준**
- [ ] 화면에 사용자 발언이 회색 말풍선으로 채팅처럼 쌓인다.
- [ ] AI가 **준비 단계에서 입력한 컨텍스트를 기억하고** 자연스럽게 받아준다 — 예: "네, 작업자 4명에 알루미늄 5052 판재 신규 자재까지 확인했습니다. 협착·면장갑 말림 사고가 과거에 있었다고 하셨으니 오늘은…"
- [ ] AI가 **사전 정보 단계**의 잔여 항목(예: 작업 시간대, 위험 인지 정도)을 한 번에 한 가지씩 묻는다.

### 3-3. 체크리스트 단계 진입

**할 일**
- AI가 묻는 질문에 음성으로 짧게 답한다(예시):
  > "오전 8시부터 정오까지 진행할 예정이고, 라이트커튼은 어제 점검했습니다."
- AI가 자동으로 5개 baseline 점검 항목을 하나씩 묻는다. 각 질문마다 음성으로 답한다:
  - "양수기동장치 정상이에요." → ✅ PR-01 체크
  - "라이트커튼 차단 시간 0.18초 확인했습니다." → ✅ PR-02
  - "금형 교체 시 개인 LOTO 락 다 가지고 왔습니다." → ✅ PR-03
  - "보호구 3종 다 착용하고 있어요." → ✅ PR-04
  - "브레이크 정지 시간 측정했습니다, 정상이요." → ✅ PR-05

**통과 기준**
- [ ] 답변할 때마다 화면 상단 게이지가 **0/5 → 1/5 → … → 5/5** 로 올라간다.
- [ ] 단계 스트립이 자동으로 **"점검"** 에 머물다, 5/5에 도달하면 **"대응"** 으로 점프한다.
- [ ] 한 단계 전환 시점에서 AI가 **누락 위험 점검**을 1회 자연스럽게 묻는다 — 예: "혹시 비상정지 위치 작업자분들이 다 인지하고 계신가요?". **사용자가 거절하면("괜찮습니다, 다음으로") AI는 즉시 양보**한다(반복 질문 없음).

### 3-4. 사진 분석으로 실시간 위험 확인 (옵션이지만 권장)

**할 일**
- 화면 하단 InputDock 우측 **카메라 아이콘** 탭.
- 처음이면 **카메라 동의 모달**이 뜸 → "허용" 클릭.
- 작업 영역(대시보드/책상 위 잡동사니, 또는 핸드폰으로 적당한 풍경)을 촬영하거나 셔터를 누른다. (실제 프레스가 없으면 대체 사진으로 OK)

**통과 기준**
- [ ] 채팅에 **(사진 첨부)** + 분석 중… 메시지가 표시된다.
- [ ] 2~5초 후 AI 요약 메시지로 교체된다 — 위험 항목 + bbox(박스 친 영역) + 신뢰도 % 가 카드 형태로 inline 노출된다.
- [ ] **신뢰도 0.7 이상 위험은 자동으로 정리본의 "주요 위험요인"에 추가**된다(우측 상단 "정리본" 버튼으로 확인 가능).
- [ ] 잘못 잡힌 항목이 있으면 카드의 **"되돌리기"** 버튼으로 정리본에서 제외할 수 있다.
- [ ] **AI가 음성으로도 vision 결과를 1~2문장 설명**한다(현재 단계가 사전정보·점검일 때만; 대응·정리 단계에선 음성 narration 생략 — Broadcast Mode 흐름 보존). auto-boost가 발동하면 "체크리스트에 자동으로 추가했습니다" 자연 언급.

> **팁** — 권한 거부 → 자동으로 "사진 파일 선택" 폴백이 뜬다. 카메라 없는 노트북에서도 갤러리 사진으로 진행 가능.

### 3-5. 상단 모드 칩 전환 동작 (옵션)

**할 일**
- 화면 상단 **"TBM"** 칩 클릭 → **"EHS"** 선택.
- 확인창 "모드 변경 시 음성 세션이 재시작되고 진행 중인 대화·체크리스트가 초기화됩니다. 계속할까요?" → **취소** (테스트에선 진행 중인 TBM을 보존).

**통과 기준**
- [ ] **취소**를 누르면 화면이 그대로 TBM에 머무르고, 누적 메시지/체크리스트가 모두 보존된다(상태 보호 동작 확인).
- [ ] (참고용 흐름 검증 — 실제로는 진행하지 말 것) 만약 **확인**을 누르면: 진행 중 TBM 세션은 IndexedDB 보존 + 신규 EHS 세션 생성 + `/ehs/<newId>` 라우트로 즉시 이동. 즉 모드 칩은 "현재 화면 mode 토글"이 아니라 **HomeScreen "EHS 안전 질문하기" 클릭과 동일한 흐름**(2026-05-07 ModeSwitcher 라우팅 fix 검증).
- [ ] 반대 방향(EHS → TBM) 검증: §6.5에서 진행. 기본 도메인 설정 유무에 따라 (a) 새 TBM 세션 + PrepareScreen 직행 또는 (b) 홈으로 이동 + 도메인 선택 시트 자동 open.

> **참고** — EHS 모드 본격 검증은 별도 시나리오로 진행한다: 홈 → "EHS 안전 질문하기"로 진입하면 자동으로 신규 EHS 세션 생성 + `/ehs/<id>` 라우트로 직행한다. EHS 챗에서 사진을 첨부하면 카드만 표시되고 "+ 체크리스트에 추가"·"되돌리기" 버튼은 미노출(체크리스트 없는 Q&A 모드라 의도된 동작). AI가 음성으로 vision 결과 1~2문장 설명 + citation 호출 시 *"관련 자료를 화면에 함께 표시했습니다"* 짧은 음성 안내 동반.

### 3-6. 정리본 미리보기 + 정리 단계 도달

**할 일**
- 우측 상단 **"정리본 NN%"** 버튼 탭 → 우측에서 미끄러지듯 슬라이드 패널이 열린다.
- 8개 필드(작업 내용 / 변화 / 위험요인 / 시나리오 / 대응 / 보호구 / 특이사항 / 참석 확인)가 보이는지 확인.
- AI에게 마무리 발언 한 번 더:
  > "오늘 마무리할게요. 작업자분들 다 안전하게 진행하시면 좋겠습니다."

- [ ] AI가 마지막에 짧은 요약(final_summary)을 생성해 정리본 상단에 보여준다.

## 10. 자주 묻는 의문 (테스트 진행자용)

**Q. AI가 사전 컨텍스트를 못 알아듣는 것 같아요.**
→ 준비 화면에서 입력 후 **"다시 받기"** 를 안 눌러서 그렇다. 입력만으로는 자동 반영이 안 된다(API rate limit 보호).

**Q. 자물쇠 아이콘이 붙은 항목이 안 사라져요.**
→ 정상이다. 카탈로그 baseline은 사용자가 끌 수 없는 필수 점검이라 의도된 동작.

**Q. 사진 분석에서 위험을 못 잡았어요.**
→ "특이 위험 미식별" 카드가 정상 노출되면 통과(거짓양성 금지가 시스템 프롬프트에 들어 있음).

**Q. PDF 한글이 깨져요.**
→ 2026-05-07 다단 fallback 적용 후 정상 동작이 표준. 브라우저에서 깨지면 **DevTools(F12) → Console** 열고 PDF 재발행 시 `[pdfGenerate]` 로그 확인:
- `Korean font embedded (subset: false) in <ms>` → 정상 (한글 정상 렌더, ~6 MB PDF)
- `embedFont(subset:false) failed ... retrying with subset:true` → 1차 throw 후 자동 폴백, 일부 글리프 손상 가능하나 ???는 아님
- `fetch returned 404` → 폰트 자산 누락 (`frontend/public/fonts/static/NotoSansKR-Regular.ttf`)
- `magic mismatch` + 첫 16바이트가 `<!do...` → SPA fallback이 index.html 반환, backend serve_spa 또는 vite 폰트 경로 점검
- `Helvetica + ASCII sanitize` → 두 embed 시도 모두 실패 → 다음 cycle에서 빌드타임 pre-subset 자산 검토 필요

**Q. 음성이 늦게 시작돼요.**
→ 자동 시작은 IndexedDB hydrate → 마이크 권한 → WebRTC 연결 → OpenAI ephemeral key 발급 순서로 ~3-5초 소요. **마이크 버튼이 부드러운 pulse 애니메이션 + "연결 중" 툴팁** 으로 시각 피드백을 제공한다(2026-05-06 회전 spinner → pulse로 교체 — 사각 테두리 회전이 어색해 자연스러운 호흡 효과로 변경).
