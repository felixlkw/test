# SafeMate 건설업 TBM 수동 테스트 시나리오

**대상**: 비개발자(현장 안전관리자, PM, 데모 진행자)
**소요 시간**: 약 20~25분 (사진 분석·PDF 다운로드 포함)
**준비물**: 노트북 또는 스마트폰 1대, 마이크, 카메라, 인터넷 연결, 백엔드 가동 중

> **테스트 콘셉트** — 가상의 신축 건축 현장에서 오늘 오전 **고소작업(외벽 마감)** TBM을 진행한다. 작업자 6명, 옥외 5층 비계 위 작업, 일기예보상 **풍속 12 m/s** 가 예상되고, 어제 인접 현장에서 공구 낙하 사고가 있었다는 설정. 이 시나리오로 앱의 모든 주요 기능(작업 선택 → 풍속 conditional 추천 → 음성 대화 → 사진 분석 → 정리·서명 → PDF 발행 → 보관함 확인)을 끝까지 사용한다.

> **건설업이 다른 도메인과 차별되는 포인트**
> 1. **풍속·날씨 conditional** — 풍속 10/15 m/s 임계값이 두 단계로 트리거되며, 외부 입력에 따라 추천 항목이 능동적으로 추가된다.
> 2. **퍼밋(PTW) 흐름** — 화기작업·밀폐공간 등은 허가서 발급이 baseline에 명시된다(History 행에 PermitChip 노출 검증).
> 3. **추락 방지 PPE** — 안전대·생명줄·안전난간이 모든 baseline에 등장 → 음성 대화에서 자연스럽게 묻고 답하기.

## 0. 시작 전 점검 (1분)

1. **백엔드 가동 확인**
   - `cd backend && uvicorn src.main:app --reload` 실행 중인지 확인.
   - `http://localhost:8000/api/health` → `{"status":"ok",...}` 응답.
2. **앱 열기**
   - `cd frontend && npm run dev` → `http://localhost:5173`.
   - 화면 상단 PwC 로고 + **SafeMate** 가 보이면 OK.
3. **권한 미리 허용**
   - 마이크 **허용**, 카메라 **허용**.

## 2. 준비 화면 — 작업 선택 + AI 위험 추천 (5분)

### 2-1. 작업 유형 선택

**할 일**
- **"작업 선택"** 섹션의 3개 카드(고소작업 / 화기작업 / 굴착작업) 중 **"고소작업"** 을 탭.

**통과 기준**
- [ ] 카드 테두리가 주황색으로 강조된다.
- [ ] 자동으로 화면 하단으로 스크롤되며 위험 추천이 1~3초 안에 채워진다.

### 2-2. 현장 컨텍스트 입력

**할 일** — 펼쳐진 폼에 다음 6개 입력:

| 필드 | 입력값 |
|---|---|
| 작업자 수 | **6** |
| 교대 | **주간** |
| 풍속 (m/s) | **12** |
| 신규 자재 | (비워둠) |
| 특이사항 | **5층 외벽 마감, 인접 현장 인접** |
| 과거 사고 키워드 | **공구 낙하, 비계 결속 풀림** |

**통과 기준**
- [ ] 풍속 12를 입력하면 conditional 트리거 임계(10 m/s)를 넘는다 — "다시 받기" 후 검증.
- [ ] AI 컨텍스트 폼이 **활성화**(disabled 아님)되어 있다(건설은 default ON).

### 2-3. AI 위험 추천 결과 + "다시 받기"

**할 일**
- 우측 상단 **🔄 다시 받기** 버튼을 1회 클릭 → 컨텍스트 반영 추천.

**통과 기준 — 필수 점검(자물쇠 🔒)**
- [ ] 5개 baseline 항목이 자동으로 펼쳐져 있다 (WAH-01~05).
- [ ] 예시: "안전대 착용 및 생명줄(수평/수직) 연결 확인", "작업발판·안전난간 설치 상태 점검", "추락방지망 설치 또는 개구부 덮개 확인", "사다리·이동식 비계 사용 시 전도 방지 조치", "공구·자재 낙하 방지(공구벨트, 낙하물 방지망)".
- [ ] 각 항목에 산업안전보건기준규칙 §42 / §13 / §14 등 법규 인용 + 출처 칩(catalog/llm).

**통과 기준 — 풍속 conditional 트리거 (핵심 검증)**
- [ ] **풍속 12 m/s 컨텍스트가 반영되어 "강풍주의 — 작업속도 제한, 신호수 추가"** 항목이 조건부 점검에 추가된다(WAH-W01, `if wind_speed >= 10`).
- [ ] 풍속을 **15** 로 바꾸고 다시 받기 시 **"강풍 작업중단(15m/s 이상) — 외부 고소작업 금지"** 가 추가된다(WAH-W02).
- [ ] 각 조건부 항목 위에 한국어 if 절(예: "if 풍속 10 m/s 이상")이 노출된다.

### 2-4. 추천 질문 확인

**통과 기준**
- [ ] 4~6개 추천 질문 칩이 노출된다.
- [ ] 예시: "오늘 안전대 생명줄 체결 직접 확인하셨습니까?", "공구벨트 스트랩 모두 일체형으로 가지고 오셨나요?", "비계 캐스터 잠금 상태 확인됐습니까?"

### 2-5. "TBM 시작" 진입

**할 일** — 화면 맨 아래 **"TBM 시작"** 버튼 탭.

**통과 기준**
- [ ] 음성 대화 화면(`/tbm/.../run`)으로 전환.
- [ ] AI 음성 세션 자동 연결(연결 중 → 대기).

## 3. 음성 대화로 TBM 진행 (8~10분)

### 3-1. 첫 응답 + 사전 정보

**할 일** — 마이크 ON 후 다음 발언:
> "오늘 5층 외벽 마감 작업입니다. 작업자 6명이고, 풍속이 12 m/s 정도로 강해요. 어제 옆 현장에서 공구 낙하 사고도 있었습니다."

**통과 기준**
- [ ] AI가 **풍속 12 m/s 를 인지하고** "신호수 추가 배치는 어떻게 하셨나요?" 같은 질문을 자연스럽게 묻는다.
- [ ] AI가 **공구 낙하 과거 사고**를 언급하며 공구벨트·낙하물 방지망을 우선 거론한다.

### 3-2. 체크리스트 단계 진입

**할 일** — AI가 묻는 5개 baseline 항목에 음성으로 답한다:
- "안전대 6명 모두 생명줄 체결 1:1 확인했습니다." → ✅ WAH-01
- "작업발판 폭 40cm 이상, 안전난간 90cm 설치 완료입니다." → ✅ WAH-02
- "개구부 덮개 다 고정했고, 추락방지망 1차/2차 설치했습니다." → ✅ WAH-03
- "이동식 비계 캐스터 4개 잠금, 아웃리거 전개 확인했습니다." → ✅ WAH-04
- "공구벨트 스트랩 일체형으로 다 지급했고, 낙하물 방지망 설치돼 있습니다." → ✅ WAH-05

**통과 기준**
- [ ] 게이지가 **0/5 → 5/5** 로 올라간다.
- [ ] 단계 스트립이 점검 → 대응 → 정리로 점프.
- [ ] 단계 전환 시점에 **누락 위험 점검**(예: "신호수 무전 채널은 통일하셨나요?")이 1회 자연스럽게 묻고, 거절 시 양보.

### 3-3. 사진 분석으로 비계·발판 점검 (권장)

**할 일** — 카메라 아이콘 → 셔터 → 작업장 사진 캡처.

**통과 기준**
- [ ] AI가 사진 내 **비계·난간·개구부** 같은 요소를 검출하고 bbox + 신뢰도 % 카드를 inline 노출.
- [ ] 신뢰도 0.7 이상은 **"주요 위험요인"** 정리본에 자동 추가.
- [ ] 잘못 잡힌 항목은 **"되돌리기"** 로 제외 가능.
- [ ] **AI가 음성으로도 vision 결과를 1~2문장 설명**(현재 단계가 사전정보·점검일 때만 활성화). auto-boost 발동 시 "체크리스트에 자동 추가했습니다" 자연 언급.

### 3-4. 화기작업 모드로 잠깐 전환 (옵션 — 퍼밋 흐름 검증)

**할 일** — 추가 발언:
> "혹시 마감재 절단 때 화기작업도 일부 있는데, 허가서 발급 절차 알려주세요."

**통과 기준**
- [ ] AI가 PTW(작업허가서) 절차, 11m 이내 가연물 제거, 소화기 2개 배치, 화재감시자 1시간 잔불 점검 등을 안내.
- [ ] 만약 LLM이 `request_permit` 툴을 호출하면 화면에 **PermitChip** 이 등장하고 History row에 표시될 수 있다.

### 3-5. 정리본 미리보기

**할 일** — 우측 상단 **"정리본 NN%"** 버튼 탭.

**통과 기준**
- [ ] 8개 필드(작업 내용 / 변화 / 위험요인 / 시나리오 / 대응 / 보호구 / 특이사항 / 참석 확인) 모두 prefill 되어 있다.
- [ ] **변화** 항목에 "과거 사고 키워드: 공구 낙하, 비계 결속 풀림" 노출.
- [ ] **위험요인** 에 baseline 5건 + 사진 자동 추가 항목.
- [ ] **보호구** 에 "전신 안전대(생명줄 일체형)", "미끄럼 방지 안전화", "안전모(턱끈 체결)", "공구벨트(스트랩 일체형)" 등.

### 3-6. (선택) 안전 경고 토스트

**할 일**:
> "지금 풍속이 18 m/s로 높아졌는데 작업 계속해도 될까요?"

**통과 기준**
- [ ] 화면 상단에 **"⚠️ 잠깐만요!"** 토스트가 즉시 뜬다.
- [ ] AI가 "풍속 15 m/s 이상은 외부 고소작업 금지입니다. 즉시 중단하세요" 같은 명확한 메시지.
- [ ] 채팅 로그에 **[안전 경고] …** 영구 보존.

## 9. 통과 / 실패 판정 요약

| 영역 | 통과 기준 | 실패 시 |
|---|---|---|
| ① 도메인 시트 | "건설" 클릭 후 준비 화면 진입 | 빌드 누락 |
| ② 풍속 conditional | 풍속 12 → WAH-W01 추가, 15 → WAH-W02 추가 | thresholds.json 미반영 |
| ③ AI 위험 추천 | baseline 5건 + 조건부 1~2건 + 출처 칩 | OPENAI 키 누락 시 catalog 폴백 — 정상 |

| ⑦ 종료 화면 | 6명 서명 + PDF 6장 분량 | 한글 폰트 미설치 시 ASCII fallback |
| ⑧ 보관함 | 도메인 "건설" + 작업유형 "고소작업" 표시 | DB v3 미적용 시 빈 row |
| ⑨ 설정 | 건설 토글 default ON 확인 | localStorage 영속 회귀 |
| ⑩ 새로고침 회귀 | 풍속 컨텍스트·conditional 보존 | IndexedDB 마이그레이션 회귀 |

## 10. 자주 묻는 의문 (테스트 진행자용)

**Q. 풍속을 12로 넣었는데 conditional 항목이 안 보여요.**
→ 입력 후 **"다시 받기"** 를 안 눌러서 그렇다. 컨텍스트 변경은 자동 fetch가 안 된다(API rate limit 보호).

**Q. 풍속 conditional 항목이 한국어가 어색해요.**
→ catalog의 `if: wind_speed >= 10` 을 한국어로 humanize하는 로직이 들어가 있지만, 임계값이 큰 편의 항목은 LLM이 다시 생성하기도 한다. 두 결과 모두 정상.

**Q. 화기작업까지 같이 다루려면?**
→ 한 세션에서 작업유형은 1개만 선택된다. 화기작업도 본격 검증하려면 새 TBM을 또 시작해서 "건설 → 화기작업" 으로 진입한다(이번 시나리오에선 음성으로 잠깐 묻기만).

**Q. 비계 사진을 찍었는데 안전대를 못 잡았어요.**
→ "특이 위험 미식별" 카드 또는 신뢰도 0.7 미만 항목은 자동 보강 안 함. 정리본에서 수동 추가 가능.

**Q. PDF 한글이 깨져요.**
→ 2026-05-07 다단 fallback 적용 후 정상 동작이 표준. 브라우저에서 깨지면 **DevTools(F12) → Console** 의 `[pdfGenerate]` 로그로 어디서 끊겼는지 진단 가능 (fetch 단계 / embed 단계 / magic 검증 / 자동 폴백 등). 자세한 분기 해석은 `manual_test_scenario_manufacturing.md` §10 FAQ 참조.
