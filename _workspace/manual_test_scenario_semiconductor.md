# SafeMate 반도체 TBM 수동 테스트 시나리오

**대상**: 비개발자(현장 안전관리자, PM, 데모 진행자)
**소요 시간**: 약 22~28분 (사진 분석·PDF 다운로드 포함)
**준비물**: 노트북 또는 스마트폰 1대, 마이크, 카메라(반도체는 default OFF — 시나리오 내에서 ON 토글), 인터넷 연결, 백엔드 가동 중

> **테스트 콘셉트** — 가상의 반도체 Fab에서 야간 **CVD 챔버 PM(예방정비)** TBM을 진행한다. 작업자 2명(2인 1조 의무), SiH4·NH3 프로세스가스 라인을 다루는 작업으로 LOTO·N2 퍼지·잔열 검증이 핵심. 인접 챔버에서 전월 PFC 노출 사례가 있었다는 설정. 이 시나리오로 앱의 모든 주요 기능 + **반도체 특유의 보안 default(컨텍스트·카메라 OFF)** 를 끝까지 검증한다.

> **반도체가 다른 도메인과 차별되는 핵심 포인트 (반드시 검증)**
> 1. **AI 컨텍스트 활용 default OFF** — 영업비밀 보호 차원에서 PrepareContextForm이 회색 처리되고 입력 자체가 막힘. 의식적으로 ON 토글 후 검증.
> 2. **카메라 사용 허용 default OFF** — InputDock 카메라 아이콘이 비활성화. ON 토글 후 검증.
> 3. **2인 1조 + EHS 입회 의무** — 단독 작업 금지 baseline (CVD-05) 명시.
> 4. **방사선·화학 가스·고압** 동시 고려 — TLD 배지·SCBA·방독마스크·내화학 보호복 PPE.

## 0. 시작 전 점검 (1분)

1. **백엔드 가동 확인** — `http://localhost:8000/api/health` → `ok`.
2. **앱 열기** — `http://localhost:5173`.
3. **권한 미리 허용** — 마이크 **허용** (카메라는 시나리오 내 토글 ON 후에 사용).

## 1. 홈 화면에서 새 TBM 시작 (1분)

**할 일**
- **"새 TBM 시작"** → 도메인 시트에서 **"반도체"** 카드 탭(힌트: "FAB·후공정·가스/화학").

**통과 기준**
- [ ] 도메인 시트가 올라온다.
- [ ] "반도체" 탭 후 **"준비"** 화면(`/tbm/.../prepare`)으로 이동.
- [ ] 도메인 표시에 **"반도체"** 노출.

## 2. 준비 화면 — 반도체 default OFF 검증 (3분)

### 2-1. 작업 유형 선택

**할 일**
- 3개 카드(CVD 챔버 PM / 가스 라인 브레이크 / 리소 장비 PM) 중 **"CVD 챔버 PM"** 을 탭.

**통과 기준**
- [ ] 카드 테두리 주황색 강조.
- [ ] 위험 추천 1~3초 안에 채워짐.

### 2-2. 컨텍스트 폼 disabled 상태 확인 (반도체 핵심 검증)

**보이는 것**
- 작업 선택 아래 펼쳐진 컨텍스트 폼이 **회색으로 비활성화**되어 있다.
- 폼 위에 안내 문구: *"반도체 도메인은 영업비밀 보호를 위해 컨텍스트 활용이 기본 비활성화되어 있습니다. Settings → 'AI 컨텍스트 활용'에서 활성화하면 입력 가능합니다."*

**통과 기준 — 보안 default 검증**
- [ ] 폼이 **disabled 상태**이며 모든 입력칸을 클릭/입력해도 반응하지 않는다.
- [ ] AI 위험 추천은 정상 작동(컨텍스트 없이 catalog seed + LLM 보강만).
- [ ] 화면 상단/하단 어디에도 사용자가 입력한 영업비밀 정보가 절대 OpenAI로 전송되지 않는 상태.

> **이 단계의 의미** — 반도체 사이트에서 바로 데모할 때 default가 OFF여서 안전팀 승인 없이도 시연 가능. 컨텍스트 폼이 활성화되어 있으면 **버그**로 간주.

### 2-3. AI 위험 추천 결과 확인 (컨텍스트 없이)

**통과 기준 — 필수 점검(자물쇠 🔒)**
- [ ] 5개 baseline 항목 자동 펼침 (CVD-01~05).
- [ ] 예시:
  - "전원·RF·가스 LOTO 시행 및 잔류전압 검전"
  - "프로세스가스(SiH4·NH3 등) 라인 퍼지 완료 — 잔류농도 측정"
  - "PFC(증착부산물) 다량 챔버 개방 시 방독마스크/SCBA 착용"
  - "히터·라이너 잔열 확인(일정 온도 이하 강하 후 작업)"
  - "PM 작업 2인 1조 및 EHS 입회 확인"
- [ ] 각 항목에 SEMI S2/F-40, 산업안전보건기준규칙 §38 등 법규 인용 + 출처 칩.
- [ ] 컨텍스트 미전송 상태라 출처 칩이 대체로 **"catalog"** 로 노출(LLM 보강 일부 포함).

### 2-4. (분기 A — 보안 우선) 컨텍스트 OFF 상태로 그대로 진행

**할 일** — **"TBM 시작"** 그대로 탭.

**통과 기준**
- [ ] 음성 화면 진입 + 자동 음성 세션 연결.
- [ ] 첫 AI 발화는 작업유형(CVD 챔버 PM)만 인지하고, 사용자 컨텍스트는 모르는 상태.

> 분기 A로 진행하면 **§3 대화 단계로 이동**. 영업비밀 절대 차단 + LLM 추천 품질 확인이 목표.

### 2-5. (분기 B — 컨텍스트 ON 검증) Settings에서 토글 활성화

**할 일**
- 좌측 상단 ← **뒤로가기** → 홈 → 톱니 아이콘 → 설정.
- **AI 컨텍스트 활용** 섹션에서 **"반도체"** 토글을 **ON** 으로 전환.
- 좌측 상단 ← 뒤로 → 홈 → "진행 중" 카드 탭(또는 새 TBM 시작 → 반도체 → CVD 챔버 PM 다시 진입).

**통과 기준**
- [ ] 토글 ON 직후 페이지 새로고침 없이 즉시 반영(localStorage 영속).
- [ ] 준비 화면 재진입 시 컨텍스트 폼이 **활성화**된다.

**할 일** — 6개 필드 입력:

| 필드 | 입력값 |
|---|---|
| 작업자 수 | **2** |
| 교대 | **야간** |
| 풍속 (m/s) | (비워둠 — 클린룸) |
| 신규 자재 | (가능하면 비워둠 — 영업비밀 우려) |
| 특이사항 | **CVD 챔버 PM, 인접 챔버 PFC 노출 1개월 전** |
| 과거 사고 키워드 | **PFC 노출, SiH4 잔류** |

- **🔄 다시 받기** 1회 클릭.

**통과 기준 — 컨텍스트 반영**
- [ ] 추천 항목·질문에 PFC, SiH4 잔류 컨텍스트가 반영된다.
- [ ] 가능하면 LLM이 동적으로 "PFC 농도 측정 + 작업시간 제한" 같은 조건부를 강화.
- [ ] 출처 칩에 **"llm"** 비율이 분기 A보다 늘어난다.

> 분기 B는 영업비밀 우려가 적은 사이트에서만 권장. 데모 진행자가 두 분기 중 하나를 골라 진행하면 된다. 두 분기 모두 통과하면 보안 게이트 + 품질 향상 두 측면을 모두 검증한 것.

### 2-6. "TBM 시작" 진입

**할 일** — 화면 맨 아래 **"TBM 시작"** 탭.

**통과 기준**
- [ ] `/tbm/.../run` 진입 + 자동 세션 연결.

## 3. 음성 대화로 TBM 진행 (10~12분)

### 3-1. 첫 응답 + 사전 정보

**할 일** — 마이크 ON 후 발언:
> "오늘 야간 CVD 챔버 PM입니다. 작업자 2명, SiH4·NH3 라인 다룰 예정이고, 인접 챔버에서 한 달 전 PFC 노출 사례가 있어서 좀 신경 쓰입니다."

**통과 기준 (분기 B)**
- [ ] AI가 **PFC 노출 과거 사례**를 인지하고 자연스럽게 첫 발화에 반영.
- [ ] AI가 LOTO 3 분야(전원·RF·가스), N2 퍼지 3회, SCBA 의무를 우선 거론.

**통과 기준 (분기 A — 컨텍스트 OFF)**
- [ ] AI는 사용자가 PFC를 직접 언급하면 그때부터 응대.
- [ ] 영업비밀 노출 위험 0(컨텍스트 미전송 상태이므로 사용자 발화만 처리).

### 3-2. 체크리스트 단계 진입

**할 일** — 5개 baseline 항목 음성 응답:
- "전원·RF·가스 LOTO 다 시행했고, 검전기로 잔류전압 0V 확인했습니다." → ✅ CVD-01
- "N2 퍼지 3회 마쳤고, 잔류농도 검지기로 SiH4 3ppm 측정했어요. 5ppm 미만이라 통과입니다." → ✅ CVD-02
- "PFC 농도 측정 후 SCBA 착용하고 진입할 계획이에요." → ✅ CVD-03
- "라이너 표면 온도 적외선으로 측정했고 45°C입니다. 50°C 미만이라 진입 가능합니다." → ✅ CVD-04
- "2인 1조 구성됐고, EHS 1회 입회 예정이에요." → ✅ CVD-05

**통과 기준**
- [ ] 게이지 0/5 → 5/5.
- [ ] AI가 SiH4 3ppm·라이너 45°C 같은 **수치 검증**을 명확히 한다("기준 5ppm 미만이라 안전합니다").
- [ ] 단계 전환 시점에 누락 위험 점검(예: "비상샤워 위치는 인지하셨나요?")이 1회 등장하고 거절 시 양보.

### 3-3. 사진 분석 — 반도체 default OFF 토글 ON 검증

**보이는 것 (default 상태)**
- InputDock 우측 카메라 아이콘이 **비활성화 또는 미노출**.

**할 일**
- 홈 → 톱니 → 설정 → **카메라 사용 허용** → **반도체** 토글 **ON**.
- 진행 중 TBM 카드로 복귀 → 카메라 아이콘이 **노출**되는지 확인.
- 카메라 아이콘 → 동의 모달 → 셔터 → 챔버 또는 대체 사진 캡처.

**통과 기준**
- [ ] **default 상태에서 카메라 아이콘이 안 보이거나 비활성화**(영업비밀 보호 + 사이트 정책 충돌 방지).
- [ ] 토글 ON 후 즉시 카메라 아이콘 활성화.
- [ ] 사진 분석 결과 카드 inline 노출(LOTO 락·검지기 같은 요소 검출).
- [ ] 신뢰도 0.7 이상 자동 보강.
- [ ] **AI가 음성으로도 vision 결과를 1~2문장 설명**(사전정보·점검 단계 한정). auto-boost 발동 시 "체크리스트에 자동 추가했습니다" 자연 언급.

> **데모 팁** — 운영팀에 보여줄 때는 **default OFF가 보안 게이트로 작동함을 강조**한다. ON 토글은 사용자가 명시적으로 결정해야 하는 행위.

### 3-4. 가스 라인 브레이크 모드로 잠깐 전환 (옵션)

**할 일** — 추가 발언:
> "오후에 가스 라인 브레이크 작업도 있는데, HF/BOE 라인 절차 알려주세요."

**통과 기준**
- [ ] AI가 **HF/BOE conditional** 인식 — "칼슘글루코네이트 젤 비치 + 작업자 사전 사용법 숙지" (GLB-HF01) 명확히 안내.
- [ ] 풀페이스 SCBA + 비상 SCBA 1조 추가 비치, 비상샤워·세안기 위치 1:1 인지를 거론.

### 3-5. 정리본 미리보기

**할 일** — 우측 상단 **"정리본 NN%"** 탭.

**통과 기준**
- [ ] **위험요인** 에 baseline 5건 + 사진 자동 추가.
- [ ] **시나리오** 에 "RF 라인 LOTO 누락 상태에서 챔버 개방 → 감전", "퍼지 미완료 상태 챔버 개방으로 SiH4 자연발화" 등.
- [ ] **대응** 에 "전원·RF·가스 3 분야 개별 LOTO + 검전기로 잔류전압 0V 확인", "N2 퍼지 3회 이상 + 잔류농도 5ppm 미만 확인 후 개방" 등.
- [ ] **보호구** 에 "절연 장갑(전기용)", "휴대용 가스검지기(SiH4/NH3 + O2)", "방독마스크 또는 SCBA", "내열 장갑" 등.
- [ ] (분기 B) **변화** 에 "과거 사고 키워드: PFC 노출, SiH4 잔류" 노출.

### 3-6. (선택) 안전 경고 토스트

**할 일**:
> "잠깐 동료가 자리 비웠는데, 혼자 챔버 안에 들어가서 작업해도 될까요?"

**통과 기준**
- [ ] **"⚠️ 잠깐만요!"** 토스트 즉시 등장.
- [ ] AI가 "2인 1조 의무, 단독 PM 작업 금지. 동료 복귀까지 대기" 명확히 안내(CVD-05 반영).
- [ ] 채팅 로그 영구 보존.

## 4. 종료 화면 — 참석자·서명·PDF (4분)

### 4-1. 종료 진입 + 요약 + 8필드

**할 일**
- 우측 상단 **"종료"** → `/finish`.
- §1 요약 검토 + 추가:
  > "CVD 챔버 PM, LOTO·N2 퍼지·잔열 50°C 미만 확인 후 진입. 2인 1조 + EHS 입회 완료."
- §2 8필드 검수.

### 4-2. 참석자 등록 + 서명

**할 일** — 2명 + EHS 입회 1명 = 3명 입력:

| 이름 | 역할 |
|---|---|
| 송재현 | 작업자 (리더) |
| 한서윤 | 작업자 |
| 김도현 | EHS |

- 각 행 ○ 서명 → 손가락/마우스 → 확인.

**통과 기준**
- [ ] 3명 모두 ● 서명 완료.
- [ ] **EHS 역할** 이 명확히 입력되어야(CVD-05 baseline 검증).
- [ ] (분기 B) 자주 함께 일하는 동료 칩에서 빠르게 추가 가능(이후 세션부터).

### 4-3. PDF + JSON 내보내기

**할 일** — §5 **"PDF 내보내기"**, **"JSON 내보내기"**.

**통과 기준**
- [ ] PDF에 도메인 "반도체", 작업유형 "CVD 챔버 PM", 3명 참석자 서명 모두 포함.
- [ ] (분기 A) JSON `prepared_context` 필드가 **undefined 또는 부재** — 영업비밀 미저장 검증.
- [ ] (분기 B) JSON `prepared_context.special_notes`에 입력값 그대로 저장(사용자 명시 ON).

### 4-4. "TBM 종료"

**할 일** — **"TBM 종료"** 탭.

**통과 기준**
- [ ] 홈 화면 자동 이동, 진행 중 카드 사라짐.

## 10. 자주 묻는 의문 (테스트 진행자용)

**Q. 반도체 사이트 정책상 카메라 자체가 금지인데도 토글로 켤 수 있나요?**
→ 앱 차원의 default OFF는 **사용자 실수 방지** 목적. 실제 사이트 정책 준수는 사용자(안전관리자)의 책임. 토글 ON 결정은 보안팀 승인 후에만 하도록 운영 가이드를 명시해야 한다.

**Q. 컨텍스트 OFF로도 LLM 추천이 잘 나오나요?**
→ catalog seed(5개 baseline) + LLM 보강이 정상 작동한다. 다만 사용자 컨텍스트가 없어 동적 조건부 항목 수는 줄어든다(보통 0~1개). 이건 **트레이드오프 — 보안 우선**.

**Q. PFC 노출 사례를 입력했는데 영업비밀 위험 없나요? (분기 B)**
→ "PFC", "SiH4 잔류" 같은 일반 화학명만 입력하면 영업비밀이라기보다 표준 안전 키워드. 다만 **공정 단계명·수율 데이터·장비 모델명**은 입력 금지. 이 부분은 운영 가이드에 명시.

**Q. SiH4 농도를 7ppm으로 답하면 어떻게 되나요?**
→ AI가 "기준 5ppm 초과 — 즉시 작업 중단·재퍼지" (CVD-G01 conditional) 안전 경고를 띄워야 정상.

**Q. 2인 1조 룰 안 지키면 어떻게 되나요?**
→ §3.6 시나리오대로 단독 작업 발언 시 안전 경고 토스트 + AI 안내. CVD-05 baseline은 자물쇠 항목이라 끌 수 없음.

**Q. EUV 리소 PM이나 가스 라인 브레이크는 어떻게 검증하나요?**
→ 한 세션에서 작업유형은 1개만 선택. 본격 검증은 새 TBM(반도체 → 리소 PM 또는 가스 라인 브레이크)으로 진입. 리소는 셔터 차단·TLD 배지·진공 챔버 압력 평형 + DUV/EUV 차광 보안경이 baseline에 등장. 가스 라인 브레이크는 이중밸브·N2 퍼지·풀페이스 SCBA가 baseline.

**Q. PDF 한글이 깨져요.**
→ 2026-05-07 다단 fallback 적용 후 정상 동작이 표준. 브라우저에서 깨지면 **DevTools(F12) → Console** 의 `[pdfGenerate]` 로그로 단계별 진단(fetch / embed / magic / 자동 폴백). 자세한 분기 해석은 `manual_test_scenario_manufacturing.md` §10 FAQ 참조.
